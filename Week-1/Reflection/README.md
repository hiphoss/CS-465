# Week 1 – Express Setup (CS-465)

In Week 1, I focused on getting the Travlr Getaways project up and running using Node.js and Express. The goal for this module was to make sure my development environment was set up correctly and that Express could successfully serve the provided static HTML pages.

I began by installing and confirming that Node.js and NPM were working, then cloned the course GitHub repository to my local machine. I created a module1 branch to keep all Week 1 work organized and used the Express generator to create the initial Express application. After installing the required dependencies, I configured Express to serve static files from the public directory.

Once the Express app was ready, I copied all of the Travlr Getaways mockup files into the public folder. This included every HTML page, along with the CSS and image assets, so the site would load correctly through the Express server.

I tested the setup by starting the server with npm start and opening the site at http://localhost:3000. The homepage and all navigation links worked as expected, confirming that the Express server was running and serving static content properly.

This folder is mainly used for organization. The working Express application and all related code changes are tracked in the module1 branch, while the required screenshot and ZIP file were submitted separately through Brightspace.
